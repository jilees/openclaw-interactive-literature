# Character Governance

## Когда создавать нового персонажа

Создавайте нового героя только если выполняется хотя бы одно:
- пользователь прямо вводит нового персонажа,
- без нового персонажа невозможно логично исполнить действие,
- новый персонаж появляется как естественное следствие сцены и остаётся релевантен >1 хода.

Не создавайте «одноразовых» персонажей без необходимости.

## Минимальная карточка персонажа

В `characters.json` для нового персонажа зафиксировать минимум:
- `name`
- `role`
- `status`

При наличии контекста добавить:
- `goals`
- `relations`
- `risks`

## Когда обновлять существующих

Обновляйте персонажа, если изменилось:
- физическое/психическое состояние,
- полномочия/роль в контуре,
- ключевые отношения,
- уровень информированности по центральной угрозе.

## Каноничность

- `characters.json` — структурный источник правды.
- `CHARACTERS.md` — читаемая сводка, синхронизируемая по необходимости.
- Если есть расхождение, приоритет у `characters.json`.
