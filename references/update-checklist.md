# Post-turn Update Checklist

Используйте после каждого успешного хода.

## Обязательно

1. `timeline.jsonl`
   - есть `commit` событие,
   - `branch_note` соответствует выбору пользователя,
   - `model` заполнено.

2. `state.json`
   - `updated_at` обновлён,
   - `phase/tension/open_loops/resolved_loops` отражают текущую сцену.

3. `episode_context.json`
   - сохранено саммари хода (~150 слов) с фиксацией концовки.

## Условно (при изменениях)

5. `characters.json`
   - добавлены/обновлены персонажи.

6. `world.json`
   - добавлены только подтверждённые изменения мира,
   - не нарушены `do_not_break`.

## Быстрая валидация

- Стык с прошлым ходом непрерывный.
- Нет внезапного реткона без объяснения.
- Текущая развилка выводится из сцены, а не добавлена формально.
